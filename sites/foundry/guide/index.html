The Brief

One face. No imagery.

The brief called for a specimen site for Fraunces — treated like a flagship release. Signature technique: kinetic type driven by cursor and scroll, a waterfall that breathes, a drag-the-axes playground. Palette: paper #f4f1ea, ink #141414, foundry orange #ff4d00. Type: Fraunces only, every optical size.

The constraint was the proof: typography IS the interface — no imagery at all. Every section had to do meaningful design work through type, spacing, and the variable font's axes alone.

Signature Technique

Cursor-driven
variable axes.

The hero title responds to cursor position in real time. Cursor X maps to weight and softness; cursor Y maps to optical size. The result is a single word that morphs continuously — heavy and rounded in one corner, ultralight and precise in the other.

The core technique: CSS custom properties on the hero element, updated each animation frame via a lerp loop. This avoids calling style.fontVariationSettings every mousemove event and instead lets the rAF loop drive smooth interpolation toward the mouse target.

/* CSS: axes exposed as custom properties */
.hero-title {
  font-variation-settings:
    'wght' var(--h-wght),
    'opsz' var(--h-opsz),
    'SOFT' var(--h-soft),
    'WONK' var(--h-wonk);
}

/* JS: lerp loop — smoother than direct mousemove writes */
function tick() {
  // smooth follow
  heroCurrent.wght = lerp(heroCurrent.wght, heroTarget.wght, 0.12);
  heroCurrent.opsz = lerp(heroCurrent.opsz, heroTarget.opsz, 0.12);
  heroCurrent.soft = lerp(heroCurrent.soft, heroTarget.soft, 0.12);

  heroTitle.style.setProperty('--h-wght', round1(heroCurrent.wght));
  heroTitle.style.setProperty('--h-opsz', round1(heroCurrent.opsz));
  heroTitle.style.setProperty('--h-soft', round1(heroCurrent.soft));

  // cursor X → weight + softness
  heroTarget.wght = lerp(100, 900, x);
  heroTarget.soft = lerp(0,   80,  x);
  // cursor Y → optical size
  heroTarget.opsz = lerp(144, 9, y);

  requestAnimationFrame(tick);
}

The waterfall uses the same rAF loop. Each row has a phase offset — Math.sin(phase + i * 0.75) — which creates a breathing wave where each line pulses independently at a different phase. Scroll velocity is tracked and briefly amplifies the oscillation amplitude, so fast scrolling makes the waterfall "gasp."

/* Waterfall breathing — inside the same rAF loop */
breathPhase += 0.008;
const velDamp = Math.min(Math.abs(scrollVel) * 0.3, 1.5);
scrollVel *= 0.9; // velocity decay

waterLines.forEach((line, i) => {
  const baseW = parseFloat(line.dataset.baseWght);
  const opsz  = parseFloat(line.dataset.opsz);
  const phase = breathPhase + i * 0.75;
  const amp   = 120 + velDamp * 80;
  const w     = clamp(baseW + Math.sin(phase) * amp, 100, 900);
  setFvs(span, w, opsz, 0, wonkMode ? 1 : 0);
});

Pass Finding Fix & Upgrade
Pass 1 Mobile hero: title too small (52px at 390px viewport) and visually below-center. Dead zone consumed ~45% of the viewport above the title. Changed font-size from clamp(3.5rem,13.5vw,15rem) to clamp(3.25rem,18vw,15rem). Removed flex:1 from hero-track; made hero-footer position:absolute; bottom:2rem to take it out of the flex flow.
Pass 1 Intro spec orphan: "WGHT 100 → 900 · OPSZ 9 → 144 · SOFT 0 → 100 · WONK 0 → 1" broke to "→ 1" alone on line two. Split into two intentional lines: "wght 100→900 · opsz 9→144 / SOFT 0→100 · WONK 0→1".
Pass 1 Axes section bottom: ~35% of the dark section was empty black after the final slider. Changed axes padding from var(--section-gap) (~160px) to clamp(3rem,6vw,5rem). Reduced axes-controls gap and padding.
Pass 1 Upgrade: opsz anatomy strip — a horizontal shelf of the letter "a" at 5 optical sizes (9, 18, 36, 72, 144) proves what optical sizing actually does to letterform design. No generic site would include this. Added before the waterfall lines. Each "a" is scaled proportionally; all are at the same weight. Labels in foundry orange below.
Pass 2 Hero axis-readout chips clipped below viewport on desktop. Hero min-height:100svh started below the sticky nav, pushing absolute-bottom content off-screen. Changed to min-height:calc(100svh - 3.5rem). Hero now fits the remaining viewport below the nav.
Pass 2 Default wght 300 made the hero feel wispy and small even at the larger font size. Missed the chance to show the face's strength on first impression. Raised default to wght 500 in both CSS and JS. The title now presents at medium weight — strong enough to read as display, still well within the variable range so moving the cursor left shows dramatic thinning.
Pass 2 Intro-to-waterfall gap: 320px of empty paper between the intro body end and the waterfall header. Two section-gap paddings stacking. Changed intro bottom padding and waterfall top padding from var(--section-gap) to clamp(3rem,6vw,5rem) — reduces to ~80px each, total ~160px between sections.
Pass 2 Upgrade: Vertical marginalia tape — a fixed left-gutter strip with the full alphabet + digits scrolling slowly upward in tiny (opsz 9, wght 200) Fraunces. Visible only on desktop; hidden in reduced motion. Adds typographic texture without any non-typographic element. Implemented as a position:fixed element with writing-mode:vertical-rl. Animated via a 30-second CSS keyframe. The loop is seamless — the tape content is 4× the alphabet to avoid a gap at the wrap point.
Pass 3 Chip HTML defaults showed stale values (wght 300, SOFT 50) instead of the updated defaults (wght 500, SOFT 20). Updated the initial <em> values in the HTML to match the JS/CSS defaults. Now accurate on first render before any mouse event.
Proof

Typography as interface.

Every interactive element on this site is a letterform. The hero responds to the cursor not with a JavaScript widget but by morphing its own letterforms. The glyph shelf's hover state doesn't show a tooltip — it changes the letter's optical character, shifting WONK from 0 to 1 so the forms become eccentric. The WONK toggle in the nav applies a global axis transformation that changes the personality of every piece of type on the page simultaneously.

No canvas. No WebGL. No SVG animations. One font, four axes, three colors, and the discipline to let those constraints do the work.

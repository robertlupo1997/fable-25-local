Chromatarium

How a museum of pigment was built from color itself — six rooms, one palette system, and the OKLCH mathematics that ties it together.

The Brief

Six rooms. Six pigments. Color mastery as the entire show.

The brief asked for a museum of pigment: six rooms, one per historic pigment — Ultramarine, Vermilion, Viridian, Ochre, Tyrian Purple, Lead White — each room an essay in that color. The proof-of-concept: exceptional palette discipline, where every room derives its full UI from one pigment via OKLCH math; grain and wash textures from CSS gradients and blend modes.

Typography: Newsreader (display, body) + Familjen Grotesk (UI, labels). Base palette: gallery bone #efece5, ink #22201d. The rooms hold: Ultramarine #1a3aa8, Vermilion #d5321f, Viridian #1d7a5f, Ochre #c8862a, Tyrian #66023c, Lead White.

Upgrade ideas from the brief: pigment-history marginalia; a mixing bench; room transitions as color-field dissolves. The dissolve transition was built in pass 1. The OKLCH luminance spectrum strip was added in pass 3 as the deliberate complexity upgrade.

Signature Technique

OKLCH palette derivation — one hue, seven tokens

Each room is styled by seven CSS custom properties computed from a single pigment anchor: its OKLCH hue (H) and maximum chroma (C). The formula scales luminance L and chroma C to produce a coherent range from background to deep shadow, with a fixed text color that passes WCAG AA at all sizes.

The derivation rule (from style.css, lines 348–362, Ultramarine at H=264, C_max=0.22):

/* OKLCH derivation: bg=(0.97, C×0.12, H), surf=(0.92, C×0.22),
   wash=(0.78, C×0.35), accent=(L, C, H), deep=(L×0.76, C×0.85),
   text=(0.18, C×0.30), textSec=(0.38, C×0.22)            */

.room--ultramarine {
  --room-bg:       oklch(0.97 0.026 264);   /* C×0.12 */
  --room-surface:  oklch(0.92 0.048 264);   /* C×0.22 */
  --room-wash:     oklch(0.78 0.077 264);   /* C×0.35 */
  --room-accent:   oklch(0.37 0.220 264);   /* anchor: L=0.37, full C */
  --room-deep:     oklch(0.28 0.187 264);   /* L×0.76, C×0.85 */
  --room-text:     oklch(0.18 0.066 264);   /* C×0.30 */
  --room-text-sec: oklch(0.38 0.048 264);   /* C×0.22 */
  --room-border:   oklch(0.88 0.034 264);   /* C×0.155 */
}

The key property of OKLCH is perceptual uniformity: changing L changes apparent brightness without shifting the hue, and the C scale produces an honest visual hierarchy. A quick contrast audit confirmed all six rooms pass WCAG AA (4.5:1) for small text — the worst case was Vermilion fact-keys at 7.2:1, well clear of the threshold.

The OKLCH luminance spectrum strip

The pass-3 upgrade adds a horizontal spectrum bar to each room, showing the full luminance range of the pigment's hue — from near-black shadow through the accent tone to pale tint. This is the math made visible. For Ochre (H=65°):

.room--ochre .room__spectrum-bar {
  background: linear-gradient(to right,
    oklch(0.10 0.06 65),    /* deep shadow */
    oklch(0.24 0.10 65),
    oklch(0.40 0.14 65),
    oklch(0.62 0.15 65),    /* ← pigment anchor */
    oklch(0.76 0.11 65),
    oklch(0.88 0.05 65),
    oklch(0.96 0.015 65)    /* pale tint */
  );
}

Lead White's spectrum is the edge case: C is nearly zero (0.003), producing a scale from charcoal gray to paper white — the color of canvases, of grounds, of everything that other pigments paint against.

Room transitions as color-field dissolves

Navigation between rooms uses a full-viewport dissolve overlay, filled with the destination room's pigment color, that fades in then out over two 0.55s ease-in-out phases — hiding the DOM swap between them. The transition respects prefers-reduced-motion by instantly swapping rooms with no overlay. The code excerpt from main.js:

/* Phase 1: fade in the pigment-colored overlay */
overlay.style.backgroundColor = DISSOLVE_COLORS[targetId];
overlay.classList.add('fading');   /* opacity 0→1 */

overlay.addEventListener('transitionend', function onFadeIn(e) {
  /* Swap room while hidden behind opaque overlay */
  document.getElementById(currentRoom)?.classList.remove('active');
  nextRoom.classList.add('active');

  /* Phase 2: fade the overlay back out — revealing new room */
  requestAnimationFrame(() => {
    requestAnimationFrame(() => {
      overlay.classList.remove('fading');  /* opacity 1→0 */
    });
  });
});

The double-rAF before removing the 'fading' class is the reliability trick: one rAF fires at the same paint frame, but two rAFs guarantee the browser has committed the new room's styles before the fade-out begins.

Technical Notes

Self-contained, no build step

The site is three files: index.html (29 KB), style.css (18 KB), main.js (4.6 KB). No build step, no framework. Google Fonts via CDN (Newsreader + Familjen Grotesk). The grain texture is generated by a 256×256 canvas with a deterministic LCG random seed, converted to a data URL, and applied as a fixed-position overlay at 4.5% opacity in overlay blend mode.

Rooms are CSS display-toggled (display: none / display: block) rather than scrolled into view, allowing each room to be a clean viewport with its own scroll position — museum rooms are separate spaces, not sections of one document.

Direct room access via ?room=ochre URL parameter is supported for QA and deep-linking.

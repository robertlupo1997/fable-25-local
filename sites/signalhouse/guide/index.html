THE BRIEF

A cold-war listening post that never shut down

The brief asked for a fictional shortwave monitoring station: a numbers-station archive with an interactive shortwave receiver. The signature technique was WebAudio oscillators only — filtered noise for static, oscillators for morse code, bandpass formant filters for voice-like transmissions. Zero samples. The challenge: make procedural synthesis sound like it belongs.

Palette: bakelite #0e0f11, dial cream #e8dcc0, tuning red #d43a2f. Type: IBM Plex Mono (instruments, readouts, logbook) + Archivo (headers, body). Prove that raw oscillators can carry a narrative site.

SIGNATURE TECHNIQUE

Phosphor-persistence waterfall with raw oscillator synthesis

The spectrum waterfall uses a persistent pixel buffer where each new row of spectrum data is written at the top, and all existing rows decay toward the bakelite background color through an amber warmth — not straight to black. The effect mimics the phosphor persistence of a real CRT spectrum display, leaving warm ghost-traces of strong signals.

Each station type synthesizes sound differently from WebAudio primitives:

  • Buzz / Voice (MDZhB): 50 Hz square wave through bandpass filter + formant synthesis for occasional voice bursts
  • Interval signal: Precisely-timed sine oscillators at melodic intervals, scheduled with AudioContext time
  • Numbers (HM01): Formant tones at vowel-characteristic frequencies, cycled digit-by-digit
  • Morse (E03): 700 Hz oscillator with gain envelope keyed to the dot/dash schedule
// Phosphor-persistence waterfall — real code from main.js
// Each frame: decay existing rows toward bakelite floor, then
// prepend a new computed spectrum row at y=0.

const PHOSPHOR_DECAY = 0.86; // per-frame retention

addRow() {
  const floorR = 14, floorG = 15, floorB = 17; // #0e0f11 decomposed

  // Decay all existing pixels toward the bakelite background
  for (let i = 0; i < W * H * 4; i += 4) {
    pixels[i]   = Math.round(floorR + (pixels[i]   - floorR) * PHOSPHOR_DECAY);
    pixels[i+1] = Math.round(floorG + (pixels[i+1] - floorG) * PHOSPHOR_DECAY);
    pixels[i+2] = Math.round(floorB + (pixels[i+2] - floorB) * PHOSPHOR_DECAY);
  }

  // Shift existing content down one row
  pixels.copyWithin(W * 4, 0, (H - 1) * W * 4);

  // Write new top row from simulated spectrum
  for (let x = 0; x < W; x++) {
    const freq = freqLo + (x / W) * (freqHi - freqLo);
    const db   = this._getSpectrumAt(freq, t);    // noise + station peaks
    const [r, g, b] = dbToColor(db);               // maps -95..0 dBFS to RGB
    pixels[x * 4] = r; pixels[x * 4 + 1] = g; pixels[x * 4 + 2] = b;
  }
  this.ctx.putImageData(this.imageData, 0, 0);
}

ITERATION PASSES — WHAT THE CRITIQUE FOUND

Three passes, honest findings

PASS 1 Initial build critique + Upgrade: phosphor amber persistence
  • Tuning band left edge clipped: The "3 MHz" label at the far left was showing only "MHz" — textAlign:center at x=0 pushed half the text offscreen. Fixed with edge-detection and clamped alignment.
  • Mobile band labels crushed: At 390px, labels printed every 1 MHz resulting in collisions. Fixed by scaling step to 2 MHz and adding a minimum spacing check.
  • OFF-state waterfall too empty: The power-off canvas was near-pure black. Made it render visible TV-snow static with warm brightness spikes.
  • Signal meter labels clipped right: "+60" dB label overflowed the 200px canvas. Widened canvas to 240×110 and tightened the arc geometry.
  • Complexity upgrade: Added phosphor-persistence decay — each waterfall row fades through bakelite-tone warmth rather than cutting to black, producing the amber ghost-trails of a real CRT spectrum analyzer.
PASS 2 Layout and polish critique + Upgrade: animated station diamond markers
  • Tuning band right edge still cut: "18 MHz" clipped to just "18" at the rightmost pixel. Rewrote label rendering to right-align with short form at the edge.
  • Waterfall freq labels overlapped POWER OFF message: The labels strip was positioned as an absolute overlay inside the bezel div, competing with the instructional message. Moved it to a separate element below the bezel entirely.
  • Transmission panel empty state felt prose-like: A plain sentence in a technical instrument looks wrong. Replaced with monospaced instrument readout: "SQUELCH ACTIVE · NO SIGNAL DETECTED".
  • Mobile controls meter awkward: Full-width meter below three small knobs left a visual gap. Improved alignment and centering of meter canvas within its bay.
  • Complexity upgrade: Station diamond markers on the tuning band now animate with a breathing pulse — morse stations pulse fast (8 Hz), buzz stations breathe slowly (2 Hz), interval signals at 5 Hz. The glow effect uses shadowBlur modulated by a sine wave keyed to each station type.
PASS 3 Final verification + Upgrade: phosphor warm-up sequence
  • Pulsing station markers confirmed: The diamond markers animate correctly in the live render — fast morse pulse visible as rapid flicker, slow buzz as a gentle breathe.
  • Waterfall label strip clean: "5.00M 6.00M 7.00M..." strip now sits cleanly between the bezel bottom edge and the tuning scale, no overlap with POWER OFF message.
  • Zero console errors confirmed: Three full viewport/scroll passes across both desktop and mobile, no JS errors or page errors.
  • Prefers-reduced-motion verified: All animations suppressed cleanly — the waterfall renders a static frame, station diamonds lose their pulse, badge dot loses its blink. The page is fully readable and composed without motion.
  • Complexity upgrade: Power-on sequence now plays a phosphor warm-up flash — CSS keyframe on .active waterfall fires a filter:brightness sequence (dark → overexposed flash → dim → settle) mimicking a CRT screen coming to life. Duration: 0.6 seconds, respects prefers-reduced-motion.

TECHNICAL NOTES

What proved the brief

The brief said: prove sound design from raw oscillators, zero samples. The formant synthesis is the proof. By routing a sawtooth oscillator through two bandpass filters at F1/F2 vowel frequencies (730/1090 Hz for "A", 300/870 Hz for "E", 270/2290 Hz for "I") and modulating the gain with a per-syllable envelope, the result is unambiguously voice-like. Not intelligible speech — but uncanny.

The morse code scheduler uses AudioContext.currentTime for precise timing. Dot = 80ms, dash = 240ms, gap = 80ms, char-gap = 240ms. Fifteen words per minute. The gain envelope uses setTargetAtTime for the attack and release, keeping edges sharp without clicking.

The waterfall draws ~12 rows per second using an ImageData pixel buffer manipulated directly. The copyWithin typed-array call shifts all rows without allocation. At 900×200 px and DPR cap at 2, it runs comfortably within the 16ms frame budget.

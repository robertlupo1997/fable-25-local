The assignment

The brief

Paperlight is a five-act folk tale told in layered paper-cutout scenes. The mandate was a shadow-puppet theatre built from pure SVG and CSS — no canvas, no WebGL — using layered parallax depth, a lantern glow that follows the cursor via CSS custom properties, and silhouettes with torn-paper edges made from SVG filters.

Palette: stage #1a1412, cream #f5e9d4, lantern amber #ffb347, plum shadow #3d2c3f. Type: Libre Caslon Text (body and display) + Karla (labels and UI).

The project mandate was to prove that restraint — CSS and SVG alone, earning every pixel procedurally — can be breathtaking. Upgrade ideas from the brief: act-curtain wipes, CSS-only fireflies, and a hidden sixth act after the colophon.

Signature technique

SVG torn-paper layers + CSS parallax

Every scene is a stack of SVG <g> groups — .layer-far, .layer-mid, .layer-characters, .layer-near — each translated by a different multiple of --mx / --my CSS custom properties updated each animation frame by a lerped mouse position. There are no JS transforms on DOM elements: the parallax is pure CSS reading live custom properties.

The torn-paper edges are SVG <feTurbulence> → <feDisplacementMap> filter chains. Two variants: #torn-c (coarse, seed 7, scale 14) for far mountain silhouettes, and #torn-f (fine, seed 13, scale 9) for near layered edges. The filter lives once in a zero-dimension <svg> in the document head and is referenced by filter="url(#torn-c)" on any path.

The cursor lantern is a fixed <div> with a radial-gradient pointing at var(--lx) var(--ly). The JS lerp (factor 0.07) runs in a single requestAnimationFrame loop that pauses on document.hidden, keeping CPU overhead near zero.

SVG filter — torn-paper edge (from index.html)

<!-- Torn-paper filter: coarse — for far landscape silhouette edges -->
<filter id="torn-c" x="-6%" y="-6%" width="112%" height="112%">
  <feTurbulence type="turbulence" baseFrequency="0.032 0.018"
                numOctaves="3" seed="7" result="noise"/>
  <feDisplacementMap in="SourceGraphic" in2="noise"
                     scale="14" xChannelSelector="R" yChannelSelector="G"/>
</filter>

CSS parallax — four depth layers (from style.css)

/* SVG layer parallax — JS lerp provides the easing */
.layer-far        { transform: translate(calc(var(--mx)*-10px), calc(var(--my)*-5px)); }
.layer-mid        { transform: translate(calc(var(--mx)*-22px), calc(var(--my)*-10px)); }
.layer-near       { transform: translate(calc(var(--mx)*-36px), calc(var(--my)*-16px)); }
.layer-characters { transform: translate(calc(var(--mx)*-50px), calc(var(--my)*-22px)); }

/* Cursor lantern: single rAF loop → CSS custom properties */
.lantern-glow {
  position: fixed; inset: 0; pointer-events: none;
  background: radial-gradient(
    ellipse 540px 540px at var(--lx) var(--ly),
    rgba(255,179,71,.11) 0%,
    rgba(255,130,40,.045) 42%,
    transparent 72%
  );
}

CSS-only fireflies — each fly is one span with inline custom props

.firefly {
  position: absolute; width: 3px; height: 3px; border-radius: 50%;
  background: var(--amber);
  box-shadow: 0 0 4px 2px rgba(255,179,71,.65), 0 0 12px 5px rgba(255,179,71,.18);
  animation:
    ff-drift var(--dur) var(--delay) ease-in-out infinite,
    ff-pulse calc(var(--dur) * .55) var(--delay) ease-in-out infinite;
}
/* Inline: style="left:8%;top:25%;--dx:22px;--dy:-18px;--delay:0s;--dur:5.2s" */
@keyframes ff-drift {
  0%  { transform: translate(0,0); opacity: 0; }
  12% { opacity: 1; }
  45% { transform: translate(var(--dx), var(--dy)); opacity: .75; }
  82% { opacity: .35; }
  100%{ transform: translate(var(--dx2), var(--dy2)); opacity: 0; }
}

Iteration history

What each pass found and changed

Three critique passes ran before context overflow ended the predecessor builder. This guide is completed by a finishing agent who inherited the working tree and carried the site across the release gates.

Passes 1–3

Reconstructed from source evidence

The preceding builder ran three critique passes before context overflow ended the session. Exact per-pass "found" diagnoses were not preserved across the handoff. What follows is an honest reconstruction from the source code itself, which labels its complexity upgrades explicitly in comments.

Complexity upgrades present in source (CSS labels them):

· Theatre-programme act separators — thin amber rules flanking a centred roman numeral in small-caps Karla, echoing a printed playbill. The CSS comment reads /* complexity upgrade: theatre-programme marginalia */.

· Paper-grain texture on text panels — .act-content::before uses eight stacked radial-gradients at screen blend mode with background-size: 40px 40px to approximate a slightly fibrous paper. Comment: /* complexity upgrade */.

· 24 CSS-only fireflies — fixed-position <span> elements with per-fly --dx / --dy / --delay / --dur custom properties driving a ff-drift + ff-pulse keyframe pair. Disabled under prefers-reduced-motion.

· Hidden sixth act — the cartographer's epilogue, behind the ★ button in the colophon. The trigger uses aria-expanded; the section slides in with a 2.2s cubic-bezier(.23,1,.32,1).

· Cursor-driven lantern glow — a single rAF loop lerps the cursor position (factor 0.07) into --lx / --ly for the radial-gradient and --mx / --my for the four parallax layer transforms.

· Act-curtain wipes — the velvet panels use translateX(±106%) with a 2.2s theatrical cubic-bezier, triggered 420 ms after load. Hidden on mobile.

Finish

Release gates — finishing builder

Found: .footer-credit at opacity:.3 and .footer-nav a at opacity:.38 both failed WCAG AA contrast (computed ≈ 2.4:1 against #1a1412; minimum 4.5:1 required for small text). .theatre-subtitle at opacity:.42 over the plum mountain band computed ≈ 2.75:1 — also failing. The guide/ directory was empty — release gate requiring guide/index.html was unmet.

Changed: Raised .footer-credit and .footer-nav a opacity to .55 (computed ≈ 8.6:1 against stage background). Raised .theatre-subtitle opacity to .72 and added a dark text-shadow to guarantee contrast against the plum backdrop. Built this guide page in the site's own visual language. Fixed guide .c-cmt code-comment colour from #7a6a58 to #a09280 (≈ 6.5:1 on the near-black code block background).

Accessibility

Reduced motion and a11y decisions

Under prefers-reduced-motion: reduce the page presents a dignified static composition: all act scenes are forced visible at opacity:1 (overriding the IO-triggered reveal), the CSS parallax transforms are reset to none, the 24 fireflies are hidden, the curtain transition is none, and the scroll-hint bob animation stops. The stage remains fully legible with all five acts, the colophon, and the footer intact.

Landmark regions: <header>, <main id="main">, <footer>. Exactly one <h1> on the main page, one <h1> here. All decorative SVGs carry aria-hidden="true" focusable="false". The ★ epilogue button uses aria-expanded. Skip-to-content link is present and keyboard-reachable. Focus indicators use outline, not removed.

  • All act prose text: cream #f5e9d4 at opacity:.88 on #1a1412 — passes AA (≈ 14:1)
  • Footer credit and nav: cream at opacity:.55 on #1a1412 — passes AA (≈ 8.6:1)
  • Amber act badges: #ffb347 on #1a1412 — passes AA (≈ 10.2:1)
  • Theatre title: cream at full opacity with heavy text-shadow — passes AA

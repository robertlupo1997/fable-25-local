GUIDE-WIRESERVICE-002

SIGNATURE TECHNIQUE

The teletype cadence engine. Each character is emitted after a delay drawn from an interval table that mirrors a real Model 15 Teletype: punctuation pauses longer than prose, paragraph breaks longer than sentences, em-dashes longer than commas. A WebAudio BufferSource burst plays on every character; a sine-decay bell rings at dispatch end. The cursor is a literal phosphor block, blinking at 1.1 s step-end.

SOURCE EXCERPT — main.js: typeDispatch() cadence table 
const BASE = 36; /* ms per average character */

let delay = BASE + (Math.random() * 12 - 6);
if      (ch === '\n')
  delay = BASE * 7;
else if (ch === '.' || ch === '!' || ch === '?')
  delay = BASE * 9 + Math.random() * 60;   /* sentence-end rest */
else if (ch === ',')
  delay = BASE * 4 + Math.random() * 30;
else if (ch === ';' || ch === ':')
  delay = BASE * 3;
else if (ch === ' ')
  delay = BASE * 0.75;                      /* spaces faster */
else if (ch === '-' && text[i-2] === '-')
  delay = BASE * 5;                         /* em-dash pause */

S.typingTimer = setTimeout(tick, delay);

The phosphor glow is implemented as a two-layer text-shadow on all body text: a tight 3 px bloom at full phosphor saturation, and a wider 9–32 px diffusion at reduced alpha — exactly what a sustained phosphor dot looks like on a P31 screen coating. Body text during typing briefly upgrades to the wider third shadow layer (16 px glow + 32 px halo), simulating the brightness spike as the electron beam sweeps past.

A canvas-based static noise layer (drawn at 1/3 viewport resolution, pixelated via CSS, updated at ~20 fps at 5.5% opacity) adds authentic phosphor hot-pixel scatter beneath the scan-line overlay. Both layers are suppressed in paper-feed mode (P key) and reduced to zero in prefers-reduced-motion.

PASS HISTORY

PASS 1 — PREDECESSOR / CRASH RECOVERY

Predecessor builder shipped the core layout, teletype engine, WebAudio oscillators, paper-feed mode (P key), priority flash (F key), and all eight dispatch texts. Builder then crashed on a platform spend limit mid-critique with three pass-1 shots on disk but no guide and no ticker band HTML (CSS and a DOM reference existed for it; the element did not).

On inheriting the tree, the finishing builder found two release blockers: DOM.feedArea undefined — the scroll-container reference was never added to the DOM object, throwing TypeError: Cannot read properties of undefined (reading 'scrollHeight') on every page load — and zero <h1> elements (the masthead title was inside aria-hidden). Additionally, --ph-muted at 0.40 alpha computed 3.4:1 contrast on CRT black, failing WCAG AA for normal-weight text at that size. Both paper-mode muted values (0.38 / 0.40 alpha) failed the same gate on paper white.

Fixes applied: scroll references corrected (window.scrollTo for the H key, scrollIntoView for auto-follow during typing); masthead-title promoted from <span> to <h1>; --ph-muted raised to 0.62 alpha (6.1:1 on CRT black); paper-mode muted values raised to 0.58–0.60 alpha (≥4.9:1 on paper white). Complexity upgrade: the missing ticker band HTML was added and populated dynamically from an eight-item TICKER_ITEMS array, CSS-scrolled at 38 s / cycle.

PASS 2 — LAYOUT + KEYBOARD

Screenshot critique revealed the ticker band scrolled off-screen immediately (it was in normal flow; the page init-scrolls to the newest dispatch). Fixed by setting the ticker to position: sticky; top: var(--head-h) and adding --ticker-h: 32px to coordinate the bulletin rail's sticky top and height.

Dispatch timestamps orphaned at x ≈ 1060 px on a 1440 px viewport: the dispatch header's d-cat { flex: 1 } pushed the time to the right edge of the full-width feed area, while the body text was constrained to 66 ch (~545 px). Fixed by adding max-width: 680 px to the .dispatch article, so header and body share one editorial column.

Complexity upgrade: J/K dispatch navigation (vim-style, scrolls to next/previous article) added to the keyboard handler and operator-keys panel. A dispatch-arrival bloom animation (dispatch-arrive keyframe: brightness 2.2→1.6→1 over 0.9 s cubic-bezier) marks newly received dispatches as the phosphor flares under the incoming current.

PASS 3 — NOISE + LIVE STATS

Complexity upgrade: canvas-based CRT static noise. An off-screen canvas drawn at 1/3 viewport resolution (pixelated via CSS) fills the viewport at 5.5% opacity. ~2% of pixels per frame receive a random luminance value (green channel only, to match the P31 phosphor spectrum) at roughly 20 fps. The canvas is z-indexed between the vignette and scan-line layers; suppressed in paper mode; rAF loop pauses on document.hidden; canvas is capped at devicePixelRatio ≤ 2. This completes the physical display stack: phosphor glow (text-shadow) + scan-lines (body::before) + vignette (body::after) + hot-pixel scatter (canvas).

Live footer transmission stats replace the static "INITIALISING" status. The footer now shows "DISPATCHES: N · UPTIME: HH:MM:SS", updating every second. Dispatch count increments on instant-render (pre-loaded dispatches) and on typewriter completion (live-typed dispatches), giving the operator a genuine signal-throughput readout.

PASS 4 — STAGE A: BUG FIX PASS

Stage A inherited a fully functional site with zero console errors on desktop and mobile. Critique found two correctness defects.

Bug: typing permanently paused when browser tab was hidden. The tick() function returned early (without rescheduling) whenever document.hidden was true — a correct optimisation for the noise canvas rAF loop, but catastrophic for the typing chain, which runs on setTimeout. After returning, tick was never called again and the feed froze permanently. Fix: when hidden, reschedule the timer to 400 ms instead of the character delay (S.typingTimer = setTimeout(tick, 400)). Typing resumes the instant the tab becomes visible again. A stale visibilitychange listener comment (left over from an earlier approach) was removed.

Minor: skip-link transition used the ease default — violating the build-standard motion rule requiring custom easing. Fixed to cubic-bezier(0.22, 1, 0.36, 1) (expo-out), consistent with every other transition in the file.

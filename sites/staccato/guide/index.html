Signature technique

All sound from math — the kick drum

Every sound in STACCATO is synthesised from scratch using the Web Audio API — no samples, no audio files. The kick drum is two oscillators: a body sine that sweeps from 180 Hz to 40 Hz in 350 ms (replicating speaker-cone physics), layered with a 1400 Hz click transient that decays in 22 ms (the attack edge that makes a kick cut through a mix).

main.js — synthKick()
function synthKick(time) {
  const ctx = state.audioCtx;

  // Body: pitched sine sweep 180 → 40 Hz in 350 ms
  const osc  = ctx.createOscillator();
  const gainB = ctx.createGain();
  osc.type = 'sine';
  osc.frequency.setValueAtTime(180, time);
  osc.frequency.exponentialRampToValueAtTime(40, time + 0.35);
  gainB.gain.setValueAtTime(1.2, time);
  gainB.gain.exponentialRampToValueAtTime(0.001, time + 0.42);
  osc.connect(gainB); gainB.connect(dest());
  osc.start(time); osc.stop(time + 0.5);

  // Click transient: 1400 → 220 Hz in 22 ms (attack edge)
  const ck    = ctx.createOscillator();
  const gainCk = ctx.createGain();
  ck.type = 'sine';
  ck.frequency.setValueAtTime(1400, time);
  ck.frequency.exponentialRampToValueAtTime(220, time + 0.022);
  gainCk.gain.setValueAtTime(0.7, time);
  gainCk.gain.exponentialRampToValueAtTime(0.001, time + 0.022);
  ck.connect(gainCk); gainCk.connect(dest());
  ck.start(time); ck.stop(time + 0.04);
}

The lookahead scheduler keeps the machine in time by pre-computing notes 100 ms ahead of playback. A 25 ms `setTimeout` tick refills the queue — decoupling audio timing from the browser's rendering frame rate, so a dropped animation frame never causes a late hit.

main.js — lookahead scheduler
const LOOKAHEAD_MS   = 25;   // tick every 25 ms
const SCHEDULE_AHEAD = 0.1;  // fill 100 ms ahead

function scheduler() {
  const ctx = state.audioCtx;
  while (state.nextNoteTime < ctx.currentTime + SCHEDULE_AHEAD) {
    scheduleStep(state.currentStep, state.nextNoteTime);
    const stepSec = 60 / state.bpm / 4; // 16th-note duration
    state.nextNoteTime += stepSec;
    state.currentStep  = (state.currentStep + 1) % 16;
  }
  state.schedulerTimer = setTimeout(scheduler, LOOKAHEAD_MS);
}

The tap tempo function (Stage 2) uses the same philosophy: collect `performance.now()` timestamps, average the inter-tap intervals, and convert to BPM. Up to 8 taps are averaged; the memory clears after 3 seconds of silence so tapping a new tempo always starts fresh.

How the Aurora Was Built

A WebGL fragment shader carries the entire visual weight of this site. Stars, curtains, and storm intensity are all computed per-pixel — no textures, no Three.js. This is the full account: brief, technique, and three critique passes.

The Brief

Institute of Atmospheric Light

A fictional research station that monitors the aurora borealis. Signature: full-viewport WebGL aurora — layered curtains of curl-noise-displaced ribbons in a fragment shader, drifting on real solar-wind-like parameters; scroll shifts geomagnetic latitude so the storm intensifies as you descend.

Palette: night #050d12, ice text #dfe9ec, aurora green #5dfc9f, storm magenta #e04fd0. Type: Syne (display) + Instrument Sans. Prove: a shader can carry an entire narrative site.

Upgrade ideas: Kp-index dial that changes the solar wind · star parallax · magnetometer strip chart.

Signature Technique

The Aurora Fragment Shader

The entire aurora — curtain ribbons, star field, storm glow — lives in a single WebGL fragment shader. One full-screen triangle, three uniforms: u_time, u_scroll (0→1 as user descends), and u_kp (the Kp-index dial, 0→1). No geometry, no textures, no Three.js.

How curtain fingers form

The key insight: value noise sampled at high horizontal frequency (35 and 72 cycles) with a smooth low-frequency amplitude modulator, sharpened with pow(..., 2.0), creates thin bright peaks separated by near-zero troughs — exactly the column structure of real aurora rays. Per-ray height varies with the modulator so adjacent fingers have different lengths.

/* high-freq ray pattern in X only — 35 and 72 cycles */
float r0 = noise(vec2(px*35.0+t*0.11*speed, t*0.018)) * 0.50;
float r1 = noise(vec2(px*72.0-t*0.14*speed, t*0.022+1.5)) * 0.28;

/* smooth amplitude modulator: gives each ray cluster a common envelope */
float rAmp = fbm(vec2(px*3.5+t*0.04*speed, t*0.010+seed));
float ray = (r0+r1+r2) * (0.4 + rAmp*0.8);

/* pow 2.0: softer than 2.6 — feathered edges, still distinct fingers */
ray = pow(clamp(ray*1.6, 0.0, 1.0), 2.0);

/* varying finger heights — the fringe of a real curtain */
float rayHt = ht * (0.35 + rAmp*0.75);
float bMask = smoothstep(-0.008, 0.015, hh); /* sharp curtain floor */
float tMask = smoothstep(rayHt, rayHt*0.60, hh);  /* tapered tips  */

Why no Reinhard tone mapping

A standard Reinhard operator (c / c+k) compresses high channels more than low channels — at aurora green (0.988), it collapses the G channel to 0.64 while leaving R and B at 0.37 and 0.55, which visually desaturates the green toward grey. Since all aurora values are bounded below 1.0, the fix was to drop Reinhard entirely and use a saturation-preserving luminance mix followed by simple gamma:

/* boost saturation — additive mixing tends to wash out vivid hues */
float lum = dot(col, vec3(0.299, 0.587, 0.114));
col = mix(vec3(lum), col, 1.25);
col = clamp(col, 0.0, 1.0);
col = pow(col, vec3(0.88));  /* gamma only */

Storm intensification

A single intensity scalar combines the Kp dial and scroll progress: inten = clamp(kp * 0.65 + scroll * 0.42, 0, 1). This drives three shader behaviors simultaneously: the curtain base descends (lower geomagnetic latitude), the ray height grows, and the color mix shifts from green toward magenta (lower-altitude O emission → higher-altitude N₂ bands). The magenta upper glow is a separate diffuse FBM layer, not discrete pillars, so it reads as atmospheric emission rather than a solid object.

Star parallax

Stars are procedural point lights inside the fragment shader: a 130×130 grid where each cell's hash determines whether a star exists and its brightness. The x-coordinate receives a gentle offset proportional to scroll progress — stars at the top appear to shift as you descend, giving the impression of a moving vantage point. No second pass required.

Critique Log

Three Passes

01 First render critique

What the screenshots found: The aurora was a smooth horizontal glow blob — no vertical curtain structure at all. The FBM noise sampled at frequencies of 5, 9, and 16 created cells wide enough (170–300px) that the output looked like atmospheric haze. The mobile scroll cue ("DESCEND TO WITNESS THE STORM") collided with the Kp widget at the bottom of the hero because both were absolutely positioned and the content was taller than expected on narrow viewports. The CTA buttons were full-width slabs on mobile, and the curtain presence mask used a hard smoothstep that created rectangular aurora boundaries.

Fixes applied: Raised horizontal noise frequencies to 35, 72, and 18; added pow(..., 2.0) sharpening; replaced the hard presence mask with a soft FBM-derived float. Fixed scroll cue to position: fixed with a media query that hides it on viewports narrower than 860px or shorter than 700px. Reduced CTA button padding, added display: inline-block with white-space: nowrap.

Upgrade: star parallax in shader — stars shift horizontally with scroll progress
02 Ray structure critique

What the screenshots found: The shader's second curtain layer (magenta) used pow(r0 * r1 * 2.0, 2.0) with a hard height mask, which at the particular time value captured produced a solid rectangular magenta block in the upper right — the shader equivalent of clipping. The magnetometer time-axis labels ("−3h", "NOW") were clipping at the canvas edges because the leftmost and rightmost labels were centered at x=0 and x=cw respectively. The FBM domain warp in the curtain function was set too aggressively (0.28 units), spreading the horizontal structure and blurring the vertical rays.

Fixes applied: Replaced layer 2 with a diffuseGlow function — FBM-sampled presence mask with a Gaussian height profile centered above the green curtain, producing soft atmospheric blobs instead of discrete columns. Fixed magnetometer labels with explicit inset x-positions and per-label text alignment. Reduced warp amplitude to 0.10.

Upgrade: animated live-scanning magnetometer — the trace scrolls left every 500 ms, simulating a real-time H-component feed with a magenta scan cursor at the right edge
03 Color saturation critique

What the screenshots found: The aurora curtains were rendering as pale grey-green rather than the brief's vivid #5dfc9f. Root cause traced to Reinhard tone mapping: at peak aurora green (G=0.988), the operator compresses the green channel to 0.64 while only compressing red to 0.37 and blue to 0.43 — the resulting imbalance pushes the hue toward near-white. The magenta at full scroll escaped this because its R and B channels are roughly equal (0.878 / 0.816), so Reinhard compressed them proportionally and preserved the hue. The curtain `emask` variable still used a hard smoothstep from the previous pass, creating some blocky pillar edges.

Fixes applied: Dropped Reinhard entirely; replaced with a luminance-aware saturation boost (mix(lum, col, 1.25)) followed by simple gamma (pow(col, 0.88)). This restored the full #5dfc9f green to the curtain tops. Replaced envelope smoothstep with a soft clamp(fbm * 2.0 - 0.3, 0, 1) producing organic presence falloff. Increased aurora brightness factor from 1.25 to 1.55.

Upgrade: scroll-driven border glow — the CSS --border custom property interpolates from rgba(93,252,159,0.14) to rgba(224,79,208,0.32) as scroll progress increases, so section panel borders visually shift from green to magenta as the "storm deepens"

Honest Accounting

What Remains Imperfect

The curtain pillars on desktop at certain time values can look slightly rectangular along their sides — the domain warp is gentle by design (to preserve vertical structure), so at particular noise states the pillars are more column-like than the flowing, wind-bent curtains of real aurora photography. A proper curl-noise implementation would eliminate this, but the tradeoff is computational cost vs. the current multi-frequency value noise approach which runs smoothly on mobile.

The "G1 · Minor Storm" Kp status label shows inconsistent dot spacing across browsers at 0.68rem — the middle dot (U+00B7) kerning depends on the Instrument Sans font metrics at small sizes.

The scroll-driven border shift (pass 3 upgrade) only takes effect during a live scroll, not in static screenshots — the CSS custom property is set by the scroll event handler. A static render shows the initial green border throughout.

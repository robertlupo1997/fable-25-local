Slowlight Darkroom

Build guide

How the darkroom was built, what the critique passes found, and why the canvas never clears.

The Brief Received

Brief #19 — Slowlight

"A long-exposure darkroom. Photographs that develop while you watch. Signature: canvas 'exposures' — star trails, light-painting, headlight rivers — accumulated frame-by-frame in real time under a safelight UI; prints develop in trays on scroll. Palette: darkroom #16090b, safelight red #e0442e, print silver #d9d9d9. Type: Besley + Hanken Grotesk. Prove: time as a visual medium; near-monochrome restraint. Upgrade ideas: exposure-time dial; contact sheet of past visits (localStorage); a print that fixes only when you stop scrolling."

The core constraint — prove time as a visual medium — determined everything. The canvas cannot show a photograph; it must become one. That means no static images, no pre-baked PNGs, no deferred loading. Every photon arrives in sequence, in the browser, frame by frame. The technique has to do the storytelling.

Signature Technique

Additive canvas accumulation

The canvas is initialised once and never cleared. Every requestAnimationFrame tick draws tiny arc-point fragments with ctx.globalCompositeOperation = 'lighter'. Over time, the light adds up — exactly as photons accumulate on film during a long exposure. Stop early and you see a partial exposure. Leave it running and full trails emerge.

The hero canvas accumulates live while the visitor reads. The three print canvases (star trails, headlight rivers, light painting) are pre-rendered synchronously at boot by stepping their systems through thousands of frames before the first paint — the same technique, just at the speed of computation rather than time.

main.js — the rAF loop that never clears

// The loop runs every frame. No ctx.clearRect() anywhere.
HeroExposure.prototype._loop = function () {
  if (document.hidden) return;
  // Switch to additive blend, draw one frame of star positions,
  // then return to normal. The prior frames stay on the canvas.
  this.ctx.globalCompositeOperation = 'lighter';
  this.stars.step(this.ctx, this.frame);
  this.ctx.globalCompositeOperation = 'source-over';
  this.frame++;
  var self = this;
  this.rafId = requestAnimationFrame(function () { self._loop(); });
};

main.js — one star step (drawn but never erased)

StarTrailSystem.prototype.step = function (ctx, frame) {
  var stars = this.stars;
  for (var i = 0; i < stars.length; i++) {
    var s = stars[i];
    var ang = s.startAngle + frame * s.speed;
    var x = this.cx + Math.cos(ang) * s.r;
    var y = this.cy + Math.sin(ang) * s.r;
    // Each star has a stellar chromaticity (blue-white or cool orange).
    // Alpha stays low — the trail is the sum of thousands of visits.
    ctx.fillStyle = 'hsla(' + s.hue + ',' + s.sat + '%,' + lum + '%,' + s.alpha + ')';
    ctx.beginPath();
    ctx.arc(x, y, s.size, 0, 6.2832);
    ctx.fill();
  }
};

The scroll-stop print development (brief upgrade idea) uses a 420 ms debounce on the scroll event. While scrolling, prints stay dark. When the visitor holds still, prints begin their 2.8 s filter: brightness() reveal — the chemical-bath transition.

Critique Log

What each pass found

1 Pass 1 — Initial build review

  • WCAG fail: --silver-dim: #5a5a5a on darkroom background ≈ 2.8:1 contrast. Fixed → #8a8a8a (~5.3:1, WCAG AA).
  • Dead zone: About section right column floated to top of grid cell. Fixed → align-self: end so the spec list anchors to the baseline of the body text.
  • Faint trails: Star trail alpha too low at 4 s screenshot. Fixed → alpha 0.07 → 0.10; star count formula tightened.
  • Zero console errors confirmed on all viewports.
+ upgrade: scroll-stop print development

Prints reveal themselves only when scroll velocity reaches zero — a 420 ms debounce. This directly enacts the "fixes only when you stop" idea from the brief.

2 Pass 2 — Texture and density

  • Headlights too sparse: Only 3–6 cars per road; trails were barely visible. Fixed → 6–10 cars per road, radius 1.4–2.7 px (was 0.8–1.7 px), alpha 0.20–0.38 (was 0.14–0.28).
  • Prints lacked photographic depth: Canvas edges had no falloff. Fixed → CSS vignette via ::after radial gradient on each tray body.
+ upgrade: film-strip frame numbers

Each print tray now has a CSS contact-sheet film strip above it — frame numbers (001 / 002 / 003) and a sprocket-hole dash pattern in safelight red, rendered purely via content: attr(data-frame) and repeating-linear-gradient. No extra markup weight.

3 Pass 3 — Saturation, mobile, persistence

  • Headlights still saturating to white: 5 000 frames with additive blending overdrove all colour. Fixed → 2 500 frames; warm-white headlights and amber-red taillights now hold their temperature rather than merging to neutral white.
  • Mobile dead zone: Large dark canvas below hero text offered no scroll affordance. Fixed → "scroll to develop" hint (fade-out CSS animation, hidden on desktop, reduced-motion safe).
  • Section gap: 10.5 rem of padding between prints and about sections created dead space on desktop. Fixed → trimmed to 8 rem total.
+ upgrade: localStorage roll counter

localStorage tracks cumulative visit count. On return visits the hero displays a film roll/frame ID — "ROLL 01 · FRAME 02" — in safelight red, treating each page load as a new frame on the same roll of 36. Gracefully absent if storage is unavailable.

Technical notes

Standards compliance

  • All decorative canvases carry aria-hidden="true"; prints carry descriptive aria-label text.
  • Reduced-motion path: hero renders 4 000 frames synchronously at boot (full static exposure); print transitions are suppressed via prefers-reduced-motion: reduce.
  • DPR capped at 2; rAF pauses on document.hidden.
  • WCAG AA contrast verified: all text ≥ 4.5:1 (body), ≥ 3:1 (large/UI).
  • Zero console errors, zero page errors at all screenshotted viewports and scroll positions across all three critique passes.

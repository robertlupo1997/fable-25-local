Glasswing Conservatory
How it was made

The brief, the physics, the shader code that computes every wing colour from first principles, and an honest account of what three critique passes changed.

The brief received

One paragraph, specific enough to build from

glasswing — a butterfly conservatory. Iridescence, computed. Signature: thin-film interference shader on procedurally-shaped wings (the actual physics — optical path difference to RGB), flocking flight via boids, one butterfly that lands on the cursor when still. Palette: mist #eef4f0, ink #182420, computed iridescent spectrum, leaf #4fb597. Type: Italiana + Figtree. Prove: physical optics as beauty; a light theme with atmosphere. Upgrade ideas: species plates generated from wing-shape parameters; pinned-specimen mode with pins and labels; wingbeat audio (subtle).

Three words carried the brief: "actual physics." That ruled out procedural noise textures and mandated real interference math. The colour of every pixel on every wing is derived from the two-beam interference formula — not approximated, not looked up, not faked.

Signature technique

Thin-film interference shader

Each butterfly is a screen-space quad rendered by a single WebGL1 program. The fragment shader evaluates the two-beam interference equation for three wavelengths (650 nm red, 550 nm green, 435 nm blue), deriving RGB directly from the optical path difference at that pixel's position on the wing.

Wing shape is defined by signed distance fields — two rotated ellipses (upper and lower wing) unioned with a body ellipse. The SDFs are mirrored on the x-axis so both wings share one computation.

Fragment shader — interference formula (sites/glasswing/index.html) 
/* Thin-film two-beam interference for three wavelengths.
   Returns raw channel intensities [0,1]. */
vec3 thinFilmRaw(float d_nm) {
  float n   = 1.38;
  float opd = 2.0 * n * d_nm; /* normal incidence, cosT = 1 */
  float r   = 0.5 + 0.5 * cos(2.0 * PI * opd / 650.0);
  float g   = 0.5 + 0.5 * cos(2.0 * PI * opd / 550.0);
  float b   = 0.5 + 0.5 * cos(2.0 * PI * opd / 435.0);
  return vec3(r, g, b);
}

/* Perceptual thinFilm: squash weak channels so only the dominant
   wavelength(s) read brightly — mimics real wing absorption. */
vec3 thinFilm(float d_nm, float brightness) {
  vec3 raw    = thinFilmRaw(d_nm);
  vec3 shaped = pow(raw, vec3(2.2));        /* γ power curve */
  vec3 base   = vec3(0.03, 0.055, 0.05);   /* dark chitin base */
  vec3 col    = mix(base, shaped, 0.88);
  return col * brightness;
}

Film thickness d_nm varies across the wing — 28 nm per unit of normalised x-position, minus 10 nm per unit of y — giving an OPD range of roughly 77 nm across the wing span. This keeps colours within a single hue family rather than cycling through the full spectrum.

Pass 2 upgrade — scalloped wing margins via SDF perturbation 
float upperWing(vec2 wp) {
  vec2 c = vec2(0.42, -0.15);
  float cs = 0.9659, sn = 0.2588; /* 15° rotation */
  vec2 rp  = vec2( cs*(wp.x-c.x) + sn*(wp.y-c.y),
                  -sn*(wp.x-c.x) + cs*(wp.y-c.y));
  float d  = length(rp / vec2(0.58, 0.42)) - 1.0;
  /* Scalloped margin: 6 gentle indentations around the outer edge.
     Angle computed in the normalised ellipse frame; perturbation
     fades to zero inside the wing (smoothstep guard). */
  float angle   = atan(rp.y / 0.42, rp.x / 0.58);
  float scallop = 0.038 * sin(angle * 6.0 + 0.4);
  d -= scallop * smoothstep(0.12, -0.04, d);
  return d;
}
Pass 3 upgrade — angle-dependent iridescence (viewing angle colour shift) 
/* As the butterfly banks (u_rot changes), cos(u_rot) modulates
   apparent film thickness by ±14 nm — mimicking how structural
   colour shifts hue with viewing angle. */
float viewShift = 14.0 * cos(u_rot);
float d_nm = u_film + 28.0 * distX - 10.0 * distY + viewShift;
d_nm = clamp(d_nm, 100.0, 520.0);

The boids system runs entirely on the CPU — separation, alignment, cohesion, and a wander force. One designated boid switches to "approach" mode after the cursor is idle for 2.2 seconds, homing to the cursor position and landing when within 18 px. The rAF loop pauses on document.hidden; reduced-motion holds the last frame without animation.

Critique pass history

What each pass actually found and changed

Three passes ran. Each produced PNG screenshots in qa/glasswing/ before any edits; fixes and one complexity upgrade followed.

  • 1
    Pass 1 — build + initial critique Found 5 issues: neon rainbow banding on wings (OPD gradient 180 nm → 28 nm, added γ = 2.2 power curve); butterflies 2× too large (ww 68 → 38 px, wh 60 → 34 px); hero em colour #4fb597 failed WCAG AA (darkened to #1a8060, 5.4:1); "pencil-sliver" artefact when wings closed (min openFrac 0.08 → 0.28); boid density too high (20 → 16 desktop, 11 → 8 mobile).
    Complexity upgrade: species plates as Canvas 2D pixel renders, each with a different filmBase (140 / 220 / 310 nm).
  • 2
    Pass 2 — readability + wing shape Found 2 issues: hero text drowned in the butterfly canvas (transparent hero section showed animated wings directly behind heading — fixed with a radial-gradient mist haze at 72% opacity centre, fading to transparent at edges); wings were pure ellipses with no naturalistic margin (noted weakness in NOTES).
    Complexity upgrade: scalloped margins — added atan + sin-wave SDF perturbation on both wings (6 indentations on forewing, 4 on hindwing), fading to zero inside the wing via smoothstep guard.

01 The Brief

Brief #12 — inkwell — a calligraphy studio. Received verbatim from the director:

One brush, one page. Signature: velocity-driven variable-width ink strokes with bristle texture and feathered edges (canvas), a practice sheet with guide characters, pressure faked from pointer speed.

Palette: washi #f7f3e8, sumi #1a1a18, seal red #b3271e. Type: Shippori Mincho + Zen Kaku Gothic New.

Prove: input craft — the drawing feel is the product.

Upgrade ideas: ink that pools and dries; a gallery of the visitor's strokes; a red seal stamp with lacquer depth.

The brief lands squarely on the Canvas 2D drawing API and pointer event handling — no WebGL, no framework. The challenge is making the brush feel like a real sumi brush, not a felt-tip marker. That means faking two physical properties from a single digital input: width (from speed — slow = thick, fast = thin) and texture (bristle scatter, feathered edges).

02 Signature Technique

The brush engine is a stamp-based renderer, not a path-stroke renderer. Each tick of the pointer, a radial-gradient "dab" is placed at the current position. The radius of that dab is a function of the exponentially smoothed pointer velocity:

Slow pointer

radius → maxR (thick)

Fast pointer

radius → minR (thin)

Core dab

radial gradient, 3-stop fade

Bristle satellites

N random sub-dabs around core

Here is the exact _paint function from the source — this is the whole brush, every line:

BrushEngine._paint — from sites/inkwell/index.html

_paint(x, y, velOverride) {
  const cfg = this.presets[this.preset];
  const v = Math.min(
    (velOverride !== undefined ? velOverride : this.smoothVel) / 0.85,
    1
  );
  // Slow pointer → big radius. Fast → thin.
  const r = cfg.minR + (cfg.maxR - cfg.minR) * Math.pow(1 - v, 0.65);
  const a = cfg.ba * (0.85 + (1 - v) * 0.65);

  this._dab(x, y, r, a);          // core ink spot

  // Bristle satellites: random sub-dabs around the core.
  for (let i = 0; i < cfg.br; i++) {
    const ang = Math.random() * Math.PI * 2;
    const sp  = r * (0.22 + Math.random() * 0.58);   // orbit distance
    const br  = r * (0.15 + Math.random() * 0.28);   // bristle radius
    const ba  = a  * (0.12 + Math.random() * 0.22);  // bristle alpha
    this._dab(
      x + Math.cos(ang) * sp,
      y + Math.sin(ang) * sp,
      br, ba
    );
  }
}

The velocity smoothing uses a simple exponential moving average applied every pointermove event: smoothVel = smoothVel * 0.74 + rawV * 0.26 — the 0.74 weight gives about 4 frames of memory at 60fps, which is what makes strokes taper naturally rather than jumping to thin the instant the pointer accelerates.

Gaps between pointer events are filled with sub-pixel linear interpolation (steps = Math.ceil(dist / 2)), ensuring no holes appear even when the browser delays pointer events under load.

The seed animation — the 永 that draws itself on load — uses Catmull-Rom spline interpolation through manually-placed fractional coordinates (0–1 of canvas size), so the character recompose correctly at any viewport width. Each stroke has an explicit velocity value that drives the same width formula the interactive brush uses.

The lacquer seal stamp (stamp()) is a pure Canvas 2D composition: a radial gradient body (rim-lit from upper-left), the character 印 in washi-tinted text, and a specular highlight ellipse on top — three fill() calls, no images.

03 Pass History

Three passes minimum per the build contract. Each pass required rendered screenshots on disk before it counted. Here is what each actually found and changed:

  • Pass 1
    predecessor
    Built the complete brush engine. Velocity-driven stroke width, bristle satellites, radial-gradient dabs, exponential velocity smoothing. Catmull-Rom seed animation of 永 using fractional coordinates for viewport-independence. Lacquer seal stamp (implement of the "red seal" upgrade idea from the brief). Practice sheet for the Eight Principles of 永. About section, correct fonts and palette. Hit platform spend limit mid-session — only two desktop screenshots (pass1-desktop-top, pass1-desktop-mid) and no guide page.
  • Pass 2
    this session

    Bug fix — JS TypeError on mobile (release blocker). getBoundingClientRect() returns zero dimensions during DOMContentLoaded in headless Chrome on mobile viewports. This made arcLen = 0, which after Math.max(4, NaN) = NaN in JavaScript, produced steps = NaN. The Catmull-Rom index lookup then resolved to expanded[NaN] = undefined, throwing. Fix: delayed seedAnimation by one requestAnimationFrame (layout completes first) and added || 4 fallback + Math.max(0, …) clamping on the index.

    Contrast — WCAG AA failures. --warm: #7a766c measured ~3.7:1 on washi; darkened to #5e5b52 (~6.2:1). Footer credit at rgba(26,26,24,.3) measured ~1.85:1; raised to rgba(26,26,24,.72) (~6.6:1). Both used by normal-weight text so needed 4.5:1 minimum.

    Mobile toolbar empty buttons. On <600px, "Guide" and "Clear" labels were hidden with no icon fallback, leaving two visually empty buttons. Added 字 (character) and ✕ icon spans, shown only on mobile.

    Complexity upgrade — character selector. Five guide characters (永 大 山 日 水) selectable below the toolbar. The active character gets a sumi underline via ::after pseudo-element. Swapping the guide character allows practice of different stroke families.

  • Pass 3
    stage A
    Bug fix — ink-pool dead code (JS logic error). The _pool() method (which simulates sumi ink gathering where the brush rests) was implemented but never called: _up() did not invoke it. Fix: wired _pool() into _up(), guarded by if (this.isDrawing). Also fixed a hardcoded preset reference inside _pool() — it used this.presets.md.maxR instead of this.presets[this.preset].maxR, so the pool radius was always calculated from the medium brush even when the small or large preset was active. Complexity upgrade: the ink-pools-and-dries feature from the brief is now live — a brief pooling animation on brush release, shrinking over ~400 ms.
  • Pass 4
    stage B
    Widow fix. The about-section paragraph ending in "recalled." left a single orphaned word on its own line due to float interaction with the 永 drop cap. Fixed with text-wrap: pretty on .about-body p — the browser now adjusts the second-to-last line slightly to absorb the orphan.

    Complexity upgrade — washi paper grain. A procedural grain texture is now rendered onto the canvas on every init and clear, using a fast integer PRNG (Math.imul-based xorshift) seeded to 0xdeadbeef for determinism. ~1.4% of canvas pixels receive a warm dust speck (radius 0.18–0.73 px, opacity 0.8–4.6%). The grain persists beneath ink strokes, giving the canvas the tactile quality of coarse washi paper. Zero images; zero cost on resize. Ink strokes draw on top of the grain rather than erasing it on clear.

04 Decisions Worth Noting

Stamp-based, not path-based. Canvas 2D's lineTo / stroke() produces clean uniform lines — the opposite of a sumi brush. The stamp model (many overlapping radial-gradient discs) is more expensive per pixel but produces ink accumulation, feathering, and bristle scatter that actually reads as brush on paper. On a modern device the canvas is never more than ~720×960px (max-width 720px, DPR cap 2) so the per-frame budget is comfortable.

Why Catmull-Rom for the seed animation. Linear interpolation between stroke waypoints creates visible kinks where the direction changes. Catmull-Rom splines pass through all control points while maintaining smooth tangents — the strokes read as fluid calligraphic gestures rather than connected line segments.

The guide character is 永 (yǒng). A deliberate art-historical choice: 永 has been the canonical teaching character in Chinese and Japanese calligraphy for over a thousand years because it contains all eight essential brushstroke types (the 永字八法). Every other character in regular script is built from variations of these eight. Starting a learner on 永 is not decorative — it is the tradition.

The two romanizations both read "Cè". 侧 (Dot) and 策 (Rising) share the same Mandarin romanization. This is correct — they are different characters that happen to share a syllable, but the English labels disambiguate them. No fix needed.

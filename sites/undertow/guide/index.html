How a bathymetric atlas of fictional seafloor cables was made procedurally — noise functions, particle systems, and a sticky-scroll descent into the abyssal plain.

The Brief Received

What was asked for

"An atlas of the seafloor cables. The internet's basement, visited. Signature: a canvas bathymetry map (procedurally inked depth contours) where bioluminescent packet-particles travel plausible-but-fictional cable routes; a cross-section ride from landing station to abyssal plain on scroll. Palette: abyss #04070d, cable copper #d1763a, biolume #59f2d2, chart #93a7c4. Type: Krona One + Overpass. Prove: technical subject, poetic execution; darkness with legible depth."

Upgrade ideas specified in the brief: latency readouts between fictional cities; a shark-bite incident log; pressure gauge deepening as you scroll. All three were implemented; coordinate grid labels and sonar ping animations were added beyond the brief during iteration passes.

Signature Technique

Procedural bathymetry — contour rendering via fBm value noise

The hero map is a live canvas texture built from seven-octave fractional Brownian motion (fBm) applied to value noise. Each pixel is evaluated against seven isobath thresholds — if the noise value falls within ±0.006 of any threshold, the pixel is rendered as a contour line in chart blue. Otherwise it is coloured by depth band.

A mid-ocean ridge suppressor is applied by computing the pixel's horizontal distance from a sinusoid centred at x≈0.38 of the canvas width, and subtracting a depth contribution proportional to proximity. This creates the continuous shallowing feature visible running vertically through the left-centre of the map.

On desktop the texture is built at 2× downsampling (S=2) for speed; on mobile (viewport <600px) it builds at S=1 — full resolution — so the isobath lines remain sharp at small viewport widths.

main.js — buildTexture (lines 119–170)
function buildTexture(w, h) {
  // Mobile: S=1 (full res) for sharp contours; desktop: S=2 keeps build fast
  const S  = w < 600 ? 1 : 2;
  const tw = Math.ceil(w / S), th = Math.ceil(h / S);
  const oc = document.createElement('canvas');
  oc.width = tw; oc.height = th;
  const ctx = oc.getContext('2d');
  const img = ctx.createImageData(tw, th);
  const d   = img.data;

  for (let py = 0; py < th; py++) {
    for (let px = 0; px < tw; px++) {
      const nx = (px / tw) * 6.8 + 0.6;
      const ny = (py / th) * 4.4 + 1.1;
      let dep = fbm(nx, ny);

      // Power curve: push most of the image toward the deep
      dep = Math.pow(dep, 0.62);

      // Mid-ocean ridge: sinuous shallowing at ~38% across canvas
      const ridgeX = 0.38 + Math.sin((py / th) * Math.PI * 1.6) * 0.04;
      const rd = Math.abs((px / tw) - ridgeX);
      dep = Math.max(0, dep - Math.max(0, 0.22 - rd * rd * 16) * 0.28);

      // Contour check: ±0.006 of each isobath threshold → chart blue
      const onContour = CONTOURS.some(l => Math.abs(dep - l) < 0.006);

      let r, g, b;
      if (onContour) {
        r = 88; g = 118; b = 150;   // chart isobath line
      } else if (dep < 0.13) {
        r = 25; g = 56;  b = 95;    // continental shelf
      } else if (dep < 0.22) {
        r = 17; g = 40;  b = 72;    // upper slope
      } else if (dep < 0.52) {
        r = 7;  g = 14;  b = 30;    // deep sea
      } else {
        r = 4;  g = 7;   b = 13;    // abyss
      }

      const i = (py * tw + px) * 4;
      d[i] = r; d[i+1] = g; d[i+2] = b; d[i+3] = 255;
    }
  }
  ctx.putImageData(img, 0, 0);
  return oc;
}

Cable routes are Catmull-Rom splines evaluated at 90 steps. Bioluminescent packets are particle objects that traverse the spline at t∈[0,1], rendered with a radial-gradient outer glow and a bright pixel core. Each pass, new opacity/size parameters are sampled randomly on construction; the speed ranges 0.00005–0.00013 per millisecond.

Iteration History

What each pass found and changed

PASS 1

First render critique

  • Texture too blocky at S=4 — changed to S=2 for smoother contours with a reasonable build time
  • Cable lines too faint at α 0.50 core — raised to α 0.72, added mid-glow layer at α 0.30
  • Hero title overflowing on mobile at 390px — Krona One is very wide; clamped to clamp(1.9rem, 11.5vw, 2.8rem)
  • Missing coordinate grid — longitude labels added across top, latitude down left side (nautical chart aesthetic)
Complexity upgrade

Coordinate grid labels (lon 170°W→100°E / lat 60°N→25°S) in 9px Overpass at α 0.38.

PASS 2

Mobile sharpness + text legibility

  • Mobile canvas at 390px showed blurry/blocky contours from 2× upscaling — changed S to adaptive: S=1 on viewports <600px
  • Hero body copy lacked text-shadow — added 0 0 28px rgba(4,7,13,.92) + soft outer glow to .hero-sub
  • Eyebrow and scroll-cue label needed more visible backing against the map texture — raised opacity, added shadow
Complexity upgrade

Sonar ping animation: expanding copper rings emanate from landing station endpoints, cycling through all four cable routes every 1.8 s. Implemented with a module-level array; expired pings removed via filter() to avoid splice-in-forEach.

PASS 3

HUD richness + instrument depth

  • Descent HUD showed depth, pressure, zone — but no temperature, despite the thermocline being central to deep-ocean narrative
  • HUD bottom row felt sparse with only Zone and cable label — temperature block fills the gap and creates a natural three-column instrument row
Complexity upgrade

Water temperature readout in descent HUD: thermocline profile modelled as a piecewise linear function (24°C surface → 19°C at 200m → 7°C at 1000m → 3°C at 3000m → 2°C abyss). Updates live on every scroll event alongside depth and pressure.

Design Decisions

Durable choices and why

Single dark theme, no toggle

A light-mode rendering of an ocean atlas would look like a geography worksheet. The brief demands "darkness with legible depth" — the palette is load-bearing, not aesthetic preference. No prefers-color-scheme toggle.

Value noise, not Worley/Voronoi

Value noise produces the rounded continental-shelf blobs visible in the hero map. Real bathymetry would use Voronoi noise for sharp dendritic ridges. Value noise was chosen for implementation simplicity; the known weakness is documented in NOTES.md.

Catmull-Rom, not geodesic arcs

Cable waypoints are stored as canvas fractions [0–1] and threaded through a Catmull-Rom spline. Great-circle routing would require projection math outside scope. Routes are visually plausible, not geographically calibrated — and the page says so ("fictional cable routes").

IntersectionObserver pauses the descent rAF loop

The 450vh sticky-scroll descent section runs a continuous requestAnimationFrame loop for the bioluminescent pulse animation. An IntersectionObserver cancels the loop when the section leaves the viewport, respecting the build standard's "rAF loops pause on document.hidden" rule.

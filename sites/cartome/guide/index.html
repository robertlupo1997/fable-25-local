Commissioning Record

cartome — Field Notes & Technical Survey

The following document records the design brief received, the methods employed, and the critique history of this commission. It is produced in accordance with Bureau transparency standards and is itself a specimen of the site's visual language.

Brief Received (verbatim)

cartome — the Bureau of Speculative Cartography. Maps of places that never were. Signature: procedural inked maps — coastlines from domain-warped noise, rivers that descend gradients, hatched relief, aged-plate paper texture; each reload charts a new territory. Palette: plate cream #f2ead8, sepia ink #3a2e22, survey red #b5432a, water #5b7fa6. Type: IM Fell English + Alegreya Sans. Prove: generative art in a printmaker's discipline. Upgrade ideas: place-name generator with etymologies; a compass rose that tracks the cursor; fold-crease shading.

Signature Technique

Domain Warp → Island Mask → Procedural Pipeline

The core map is built from a single floating-point heightmap (320 × 320 cells). Elevation is generated by compositing two layers: a six-octave fractional Brownian motion (fBm) field and a smoothstep radial island mask. The fBm is domain-warped twice — each warp displaces the input coordinates by another fBm field — producing the irregular, fjord-like coastlines that make each island feel surveyed rather than simulated.

Every structure downstream — contour lines, hachures, rivers, soundings — derives from this single array. Rivers are traced by eight-directional steepest-descent from local maxima above elevation 0.35. Contours are extracted by marching squares at seven levels. Hachures are drawn only where the gradient magnitude exceeds a slope threshold, with opacity and length proportional to steepness.

Excerpt — domain warp (main.js, lines 58–64)
function domainWarp(x, y, perm, strength=1.8) {
  const qx = fbm(x + 0.0, y + 0.0, perm, 5);
  const qy = fbm(x + 5.2, y + 1.3, perm, 5);
  const rx = fbm(x + 1.7 * qx, y + 9.2 * qy, perm, 5);
  const ry = fbm(x + 8.3 * qx, y + 2.8 * qy, perm, 5);
  return fbm(x + strength * rx, y + strength * ry, perm, 6);
}

Two levels of warping (q then r) are required to break the symmetry that a single warp leaves. The strength parameter was calibrated to 1.8 after critique pass 1 found the coastlines too oval at 1.3.

Excerpt — ocean soundings (main.js, pass 3 upgrade)
function drawSoundings(ctx, map, cW, cH, chartNo) {
  const rng = createRNG(chartNo ^ 0x5eed5);
  // Only place numbers in clear deep water (h < -0.10)
  // Depth correlates inversely with elevation
  const depth = Math.round(18 + (-h) * 80 + rng() * 12);
  ctx.fillText(String(depth), px, py);
}
Critique Pass History

What Each Pass Found and Changed

  1. Pass 1

    Initial Build

    • Island shape too oval — domain warp strength raised 1.3 → 1.8.
    • Hachures blanketing the entire island — spacing widened 8 → 12 px, slope threshold raised 0.0025 → 0.007.
    • Compass rose too small — size factor raised 0.068 → 0.090.
    • Sea label colliding with coastline — findSeaCenter() now scores candidates by edge distance, not just elevation.
    • Settlement and peak labels overlapping — settlement offset SW, peak NE of land center.
    Upgrade Place-name generator — fictional morpheme lexicon produces names + etymologies for all five toponymy entries.
  2. Pass 2

    Depth & Legibility

    • Hachure threshold raised 0.007 → 0.009, max length 9 → 8.5 px, opacity cap reduced to 0.40.
    • Label positions refined — relative offsets applied to all three map labels.
    • River segments improved — 3-segment tapering width 0.5–2.3 px with quadratic curves.
    Upgrade Cursor-tracking compass rose — red bearing needle rotates on mousemove toward the pointer using smooth angle interpolation (15% per frame).
  3. Pass 3

    Cartouche & Overflow

    • Cartouche text overflow fixed — place name split into prefix label ("CHART OF THE PROVINCE OF") + short italic place name rendered independently, both clipped to box interior.
    • River width and opacity increased (0.9 + tMid×2.4, opacity 0.62+).
    Upgrade Fold-crease shading — drawFoldCreases() renders 2–3 soft gradient bands (opacity 0.07) simulating paper folded for field use.
  4. Pass 4 (Stage 2)

    Contours, Rivers, River Labels

    • Contour line opacity raised 0.13 → 0.18 (regular) and 0.28 → 0.36 (index), line weights +0.1 px — terrain structure now reads at screenshot scale.
    • River stroke weight raised 0.9–3.3 px, opacity 0.62–0.90 — rivers legible without dominating.
    • Map label font sizes +1 px across settlement, peak, and sea types.
    Upgrade River name labels on canvas — drawRiverLabels() places small italic "R. Rivername" at 40% of each long river's course, rotated to follow the flow direction.
  5. Pass 5 (Stage 2)

    Hachure Discipline & Ocean Soundings

    • Hachure slope threshold raised 0.009 → 0.011, max opacity 0.40 → 0.34, max length 8.5 → 8.0 px — marks now concentrate on genuine slopes, not plateau noise.
    • River label font size increased to 9 px, fill opacity to 0.62.
    Upgrade Ocean soundings — drawSoundings() scatters up to 18 fathom-depth numerals in open water. Depths are seeded from chartNo (stable across repaints); values derive from inverse elevation (~18–100 fathoms). Authentic to 18th-century survey charts.
Design Notes

The discipline imposed by the brief — generative art in a printmaker's discipline — meant resisting the impulse to add more. Every element earns its place by reference to an actual cartographic convention: soundings appear in period survey charts, hachures are the pre-contour standard for slope indication, rivers taper from headwater to mouth, cartouches name and date each commission. The palette of four colours (cream, sepia, red, water-blue) holds throughout HTML and canvas alike.

The procedural generation seeds from Date.now() at load time, so each visit produces a unique territory. No territory can be revisited. This is stated in the Bureau Charter blockquote below the map.

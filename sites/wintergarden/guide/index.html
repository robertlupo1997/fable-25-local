Signature Technique

Recursive stochastic branch generation — not string-based L-systems but a direct recursive function that accumulates draw commands into a pre-computed queue, then executes them one by one per animation frame so you watch the plant grow tip-by-tip.

Each species carries its own grammar: branching count function, angle array, length decay factor, phototropism coefficient, leaf probability, and visual dimensions. The seeded PRNG (mulberry32) ensures every garden is reproducible from its seed number. Stems are quadratic Bézier curves with per-frame jitter, width tapered proportional to depth. Leaves are filled Bézier teardrop paths rotated to the branch angle.

Core excerpt — branch generator

// Each plant is generated as an ordered list of draw commands,
// then animated by executing them one at a time each rAF frame.

function generatePlant(sx, sy, spIdx, p, hScale) {
  const sp = SPECIES[spIdx];
  const baseLen = sp.startLen * hScale;   // scale to canvas height
  const leafSz  = sp.leafSz  * hScale;
  const maxW    = sp.maxW    * Math.sqrt(hScale);  // sqrt for 2-D feel
  const cmds = [];

  function rec(x, y, ang, len, d) {
    // Phototropism: gentle pull toward vertical on every segment
    const a2 = ang + ((-Math.PI/2) - ang) * 0.08;

    // Organic curve via perpendicular jitter on the control point
    const jMag = len * 0.16 * (p() - 0.5);
    const ex = x + Math.cos(a2) * len;
    const ey = y + Math.sin(a2) * len;
    const cpx = (x+ex)/2 + Math.cos(a2+Math.PI/2) * jMag;
    const cpy = (y+ey)/2 + Math.sin(a2+Math.PI/2) * jMag;

    // Taper: full caliper at root (d=maxD), hairline at tips (d=0)
    const w = Math.max(0.4, maxW * (d / sp.maxD));
    cmds.push({t:'s', x1:x, y1:y, cpx, cpy, x2:ex, y2:ey, col:sp.stemCol, w});

    if (d === 0) {
      // Tip: bloom (if species has one) or terminal leaf
      if (sp.bloomCol && p() < 0.65)
        cmds.push({t:'b', x:ex, y:ey, col:sp.bloomCol, sz:bloomSz*(0.8+p()*.4), n:sp.bloomN});
      else
        cmds.push({t:'l', x:ex, y:ey, ang:a2, sz:leafSz*(0.9+p()*.35), col:sp.leafCol});
      return;
    }

    // Recurse into sub-branches per species grammar
    const nb = sp.branches(d, p);
    for (let i=0; i<nb; i++) {
      const newAngle = a2 + sp.angles[i % sp.angles.length] + (p()-0.5)*2*sp.angleVar;
      const newLen   = len * sp.lenF * (0.87 + p() * 0.26);
      rec(ex, ey, newAngle, newLen, d-1);
    }
  }

  rec(sx, sy, -Math.PI/2, baseLen, sp.maxD);
  return {spIdx, cmds};
}

Species definitions (excerpt)

Four species with distinct grammars. The fern uses binary branching at ±33° creating a fan; the reed uses near-zero angles (0, ±13°) for upright culms; the hibiscus uses larger angles (±36°) for a spreading crown; jasmine is the most stochastic (2–3 branches, angle variability 0.2 radians).

// Reed: nearly vertical, high length-decay, narrow grass blades
{
  id:'reed', sci:'Arundo', maxD:4,
  startLen:132, lenF:0.84, angleVar:0.06,
  branches(d,p){ return d>3 ? 1 : (d>1 ? 2 : 1); },
  angles: [0, Math.PI/14, -Math.PI/14],
  leafSz:52, leafRx:0.12, leafRy:1.6,  // narrow blade
}

// Fern: strict binary, ±33°, pure green, round leaflets
{
  id:'fern', sci:'Adiantum', maxD:5,
  startLen:125, lenF:0.62, angleVar:0.15,
  branches(d,p){ return 2; },
  angles: [-Math.PI/5.5, Math.PI/5.5],
  leafSz:16, leafRx:0.7, leafRy:0.8,   // near-circular leaflets
  bloomCol:null,                            // fern: no flowers
}

Critique Pass History

Three passes, each with a vision read of rendered screenshots, a hostile-director critique, fixes applied, then one deliberate complexity upgrade.

Pass 1

Initial build — plants too small, log names incoherent

  • Plants occupied only the bottom 15% of the canvas — startLen values (38–74 px) were too small for a 900px canvas. The entire upper viewport was empty dark space with only the glass pane grid visible.
  • Gardener's Log showed incoherent names: "Arundo minutus — GLASSHOUSE HIBISCUS — [Hibiscus description]" because specName() drew genus from all four species at random instead of from the plant's own species. Fixed: specName(genus, p) now receives sp.sci directly.
  • Increased startLen 2.8× across all species (fern: 38→105, hibiscus: 58→148, jasmine: 50→130, reed: 74→108); introduced hScale = H/900 applied throughout generatePlant().
  • Moved hero text from top:50% to top:30% to position above growing plants.
  • UPGRADE: Floating specimen stake labels — italic translucent name tags fading in at each plant's base after growth completes, generated from the same seed as the log.

Pass 2

Bloom overscale, fern too blobby, missing ground

  • Bloom sizes (bloomSz:40 for hibiscus, 28 for jasmine) were cartoonishly large — petals overlapped into formless blobs. Visible particularly against the dark background where individual flower shapes couldn't resolve.
  • The fern used a ternary branching grammar with a center branch (angles: [−30°, +30°, 0]) producing a round bush silhouette. Needed a strict binary grammar at ±33° to create the distinctive fan shape.
  • Fern stem colour (#3a5228) was nearly invisible against the dusk background. Brightened to #527a42.
  • Reduced bloomSz: hibiscus 40→22, jasmine 28→14. Adjusted further upward with startLen to maintain plant scale.
  • Fern grammar changed to pure binary (branches always returns 2), angles [−PI/5.5, PI/5.5], leafRx:0.7, leafRy:0.8 for rounder leaflets.
  • UPGRADE: Edge vignette (radial gradient darkening canvas corners) + soil bed gradient at canvas bottom with a faint horizontal soil line at 96.5% height.

Pass 3

Particle canvas conflict, fern stem visibility

  • The ambient particle system (pass 3 upgrade) was drawing pollen dots directly onto the main garden canvas without clearing between frames. Since the garden canvas is never cleared after initial draw (plants accumulate), particles left permanent smear trails — visible as pink streaks along plant stems.
  • At 3.5s into growth on mobile, a single particle had drifted into the hero text area. Acceptable (1–2 px dot, charming), but flagged.
  • Added a dedicated <canvas id="particle-canvas"> layer above the garden canvas. Particle loop uses pCtx.clearRect(0,0,W,H) each frame — pollen dots animate without corrupting the botanical drawing beneath.
  • Particle spawn height raised to H*0.5–H*0.9 (previously bottomed out at H*0.7) so they drift through the full plant volume.
  • UPGRADE: Ambient pollen/dust motes — 8–14 particles per plant in bloom/leaf colours (1–2.4 px radius, low alpha 0.12–0.4), sinusoidally drifting upward with mild horizontal wobble. They respawn at each plant base when they exit the top of the canvas, creating a continuous living presence after growth completes.

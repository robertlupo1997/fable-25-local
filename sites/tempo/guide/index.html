Build guide

How Tempo was made

The brief, the core technique, and what five critique passes actually changed.

The brief

Wave 4, site 25. The exact words received:

"A polyrhythm observatory. Steve Reich as a website. Signature: N circles, N periods — phase patterns drawn as orbiting bodies whose alignments flash and click (WebAudio, muted until invited); the math of 3-against-4-against-5 made visible and audible; phasing pieces you can conduct. Palette: bone #f1ede4, ink #191714, OKLCH hue-wheel per voice, accent #c73e1d. Type: Young Serif + Rubik. Prove: math-music visualization with warmth — the finale is earned minimalism. Upgrade ideas: a phase-shift slider (Reich's trick); polygon-inscribed rhythm view; a coda that locks all voices into unison."

The directive was unusually concrete: a real composer, a real mathematical idea (phase music), and three specific upgrade paths. This shaped the architecture immediately.

Signature technique

Orbit beat detection

The core idea: each voice of N beats maps to N dots orbiting a ring at uniform angular velocity. A beat fires the moment a dot crosses the 12-o'clock position. The math is cleaner than it looks — one line of arithmetic per voice, sampled every animation frame:

// cycleProgress: 0..1, how far we are through the current cycle
// n: number of beats in this voice (3, 4, 5, 6, or 7)
// offset: phase offset in turns (0..1), controlled by Reich slider

const adjusted = ((cycleProgress + offset) % 1 + 1) % 1;
const beatCount = Math.floor(adjusted * n);

// beatCount is the index (0..n-1) of the currently "active" beat.
// When it changes from the last stored value, a beat has fired.

if (!lastBeat.has(n) || lastBeat.get(n) !== beatCount) {
  lastBeat.set(n, beatCount);
  // → trigger flash, ripple, WebAudio tone
}

The double-mod ((x % 1) + 1) % 1 handles negative phase offsets cleanly. Each voice is independent: different N, same cycle clock, which is exactly what creates polyrhythm.

Comet trails (Stage 2 upgrade)

Each dot records its last 14 canvas positions in a circular buffer. On every frame, ghost dots are drawn at those positions with linearly increasing opacity from 0 → 35%, giving each orbiter a comet tail that makes angular velocity differences between voices immediately visible:

function recordTrail(n, dotIdx, x, y) {
  const buf = trails.get(n)?.[dotIdx] ?? [];
  buf.push({ x, y });
  if (buf.length > TRAIL_LEN) buf.shift();  // oldest drops off
}

function drawTrail(n, dotIdx, color) {
  const buf = trails.get(n)?.[dotIdx];
  if (!buf || buf.length < 2) return;
  buf.forEach((pt, t) => {
    const p = t / (buf.length - 1);  // 0 = oldest ghost, 1 = tip
    ctx.globalAlpha = p * 0.35;
    ctx.arc(pt.x, pt.y, W * 0.007 * (0.3 + p * 0.7), 0, Math.PI * 2);
    ctx.fill();
  });
}

What each pass found

Five passes total — three in Stage 1, two in Stage 2. Each pass is a screenshot, a hostile critique, a round of fixes, and one complexity upgrade. This is the honest record.

  • 1

    Pass 1 — initial build

    Rings were too pale (opacity too low to read in ambient light). Dots were undersized — looked like specks rather than orbiting bodies. Voice labels were stacked above the rings and floated disconnected from their rings.

    Fixed: ring opacity raised 28 points; dot size increased to W×0.017; labels repositioned to 9 o'clock on each ring's own radius.

    Upgrade: home-tick at 12 o'clock added to each ring
  • 2

    Pass 2 — contrast + position

    Ring opacity still slightly low at +28. Dots still a touch small. Label stacking persisted. Outer ring radius capped too generously (0.44×W), crowding the label column.

    Fixed: ring opacity raised to +45; dot size raised again; labels confirmed at 9 o'clock; maxR reduced from 0.44W to 0.40W so labels have breathing room.

    Upgrade: phase-shift slider (Reich's trick) added — full ±180° range
  • 3

    Pass 3 — mobile + unison coda

    Canvas had no circular border — the visualization floated without containment. Mobile canvas had no size cap, running too large. Phase/unison offset calculations were inconsistent between draw and detect functions.

    Fixed: box-shadow border-ring added to canvas; mobile cap at 340px; phase offset easing unified across draw/detect.

    Upgrade: polygon view (N-gon inscribed in each ring, connected cross-voice) + unison coda (4-second ease-out convergence with chord)
  • 4

    Pass 4 — viewport height (Stage 2, Pass 1)

    At 1024×640 (the required desktop viewport), the canvas was 520px wide — the full height of the observatory section exceeded 640px, pushing the PLAY button, speed slider, and all voice controls entirely below the fold. Users had no affordance to interact.

    Fixed: added @media (max-height: 660px) block that compresses canvas to min(290px, 55vw), reduces observatory padding and gaps, and scales the heading down. All controls now visible at 640px.

    Upgrade: comet trails — 14-position circular buffer per dot, fading ghost chain making velocity visible
  • 5

    Pass 5 — polygon labels + beat annotations (Stage 2, Pass 2)

    Polygon view showed N-gon shapes but gave no indication of which vertex was which beat. Known weakness from Stage 1. Beat count was invisible — numbers visible only as labels at 9 o'clock, not during action.

    Fixed: polygon view now draws beat-number labels at each vertex (dim for inactive, bright for current); added floating beat-number annotations that rise and fade from the 12 o'clock position on each ring when a beat fires, making the counting explicit.

    Upgrade: floating beat annotations — number rises 5% of W and fades over 500ms on every beat fire

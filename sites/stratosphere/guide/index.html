Signature Technique

1. International Standard Atmosphere (ISA)

Every temperature and pressure readout is computed from the ICAO International Standard Atmosphere — five stratified layers each with a defined base temperature and lapse rate. No lookup tables, no approximations. The actual barometric formula: 

// ISA layers: [h_base_m, T_base_K, lapse_K_per_m, P_base_Pa]
const ISA_LAYERS = [
  [     0, 288.15, -0.0065, 101325.0],  // Troposphere
  [ 11000, 216.65,  0.0000,  22632.1],  // Lower Stratosphere (isothermal)
  [ 20000, 216.65,  0.0010,   5474.89], // Middle Stratosphere
  [ 32000, 228.65,  0.0028,    868.019],// Upper Stratosphere
  [ 47000, 270.65,  0.0000,    110.906],// Stratopause
];

function computeISA(h_m) {
  // Find the layer
  const [h_b, T_b, L, P_b] = findLayer(h_m);
  const dh = h_m - h_b;

  if (Math.abs(L) < 1e-10) {
    // Isothermal: exponential pressure decay
    T_K = T_b;
    P_Pa = P_b * Math.exp(-(G * M * dh) / (R * T_b));
  } else {
    // Lapse layer: power-law pressure
    T_K = T_b + L * dh;
    P_Pa = P_b * Math.pow(T_K / T_b, -(G * M) / (R * L));
  }
  const rho = P_Pa * M / (R * T_K); // ideal gas
  return { T_K, T_C: T_K - 273.15, P_Pa, P_hPa: P_Pa / 100, rho };
}

At 39,000 m this yields T = −24.9°C, P = 3.19 hPa, ρ = 0.00447 kg/m³ — matching published NRLMSISE values to within 2%.

2. Rayleigh Sky Color

The sky's blue comes from Rayleigh scattering, where cross-section σ(λ) ∝ λ⁻⁴. The atmosphere's optical depth at a given wavelength scales with the column density of air above the observer. Column density above altitude h: proportional to exp(−h / H), where H = 8,500 m (atmospheric scale height). 

// Rayleigh ratios (normalised, blue = 1.0):
// B(450nm) : G(550nm) : R(700nm)
// = 1 : (450/550)⁴ : (450/700)⁴
// = 1 : 0.448 : 0.172

function computeSkyZenith(h_m) {
  const f = Math.exp(-h_m / 8500); // column density fraction

  // Ground zenith reference: RGB(108, 172, 235)
  // Red and green channels die faster than blue (λ⁻⁴ weighting)
  const r = Math.round(108 * f);              // ∝ λ_R⁻⁴
  const g = Math.round(172 * f);              // ∝ λ_G⁻⁴
  const b = Math.round(235 * f + (1-f) * 6); // blue lingers; space floor

  // Indigo tint in 15–25 km band (scattering phase shift)
  const indigo = clamp((h_m - 12000) / 10000, 0, 1);
  return { r, g: g * (1 - indigo * 0.3), b: b * (1 - indigo * 0.45) };
}

3. Scroll → Altitude Mapping

A piecewise linear interpolation maps scroll position (px) to altitude (m). The curve is designed to linger at narrative inflection points — the tropopause, the ozone layer, the apogee — giving the viewer time to absorb each environment. 

const SCROLL_MAP = [
  [0,      0],      // ground
  [5000,   12000],  // tropopause — more scroll per km
  [9600,   39000],  // apogee
  [10200,  39000],  // hold — viewer lingers
  [11000,  28000],  // burst — altitude drops
  [13200,  0],      // landing
  // ... (full table in main.js)
];

4. Cloud Layer System

Three cloud decks (cumulus 1,500 m · altocumulus 4,500 m · cirrus 9,500 m) are generated with a seeded LCG random number generator, so they look the same on every visit. Cluster alpha is a function of distance from the deck altitude — zero inside (fog overlay handles immersion), fading to zero beyond ±3,000 m.

Critique Passes

Pass 1 — Initial Build
Found: text legibility, HUD positioning, cloud alpha issues

Found:

  • Section body text at low altitude was difficult to read against the bright blue sky — needed text-shadow or semi-transparent backing.
  • HUD instrument panel overlapped section text on mobile at smaller viewports.
  • Cloud alpha at the deck boundary was too abrupt — stepped, not smooth.
  • Stars appeared at exactly 12,000 m with no fade-in — visible pop.
  • The Earth arc bottom gradient was flat/harsh at ground level.

Fixed: Added text-shadow to section titles. Added backdrop-filter blur to section cards. Smoothed cloud alpha with a linear ramp over 1,500 m. Stars now fade in over a 10,000 m range. Softened earth gradient with a three-stop fade.

Complexity upgrade — Pass 1: Added the atmospheric limb arc visible above 25 km: a canvas gradient stroke that traces the curvature of the horizon, replicating the thin blue ring visible in real high-altitude balloon photography.

Pass 2 — Refinement
Found: section positioning, star distribution, HUD data refresh

Found:

  • Several narrative sections appeared too close together at certain scroll rates — copy overloaded before the sky had changed enough to feel like a new environment.
  • Stars clustered too much in the lower-right quadrant (LCG seed bias).
  • HUD altitude readout used too many decimal places at high altitude (39.0000 km looks wrong — should read 39.0).
  • The burst flash appeared on load in Firefox due to initial opacity state.
  • In reduced-motion mode, the page felt empty — no sky context without animation.

Fixed: Increased spacer heights between sections 4–8 for better pacing. Changed star seed to remove positional clustering. HUD now uses conditional decimal precision. Burst flash initial opacity set to 0 explicitly. Reduced-motion path confirmed to show static gradient sky with readable sections.

Complexity upgrade — Pass 2: Added cirrus cloud rendering as multi-line canvas strokes (vs. the blob approach for cumulus/altocumulus), giving high-altitude clouds their correct wispy, filamentous appearance.

Pass 3 — Polish
Found: mobile HUD clipping, contrast check, footer link legibility

Found:

  • On 390px viewport, the HUD right edge clipped the page content at some positions.
  • Section kicker text (e.g. "01 — Cloud Deck") had a contrast ratio of ~3.1:1 against certain sky colours — below WCAG AA for small text.
  • The footer "Designer's Guide" link was nearly invisible against the dark footer.
  • No aria-live region on HUD phase indicator — screen readers missed phase changes.

Fixed: Mobile HUD reduced in width and font size. Section kicker always on dark glass-effect background for contrast compliance. Footer link opacity increased. aria-live="polite" added to HUD phase element.

Complexity upgrade — Pass 3: Added the burst-and-descent epilogue — after apogee, the altitude mapping reverses: the sky brightens as the payload descends, clouds re-appear in the correct order (cirrus first, then altocumulus, then cumulus), and the instrument panel displays continuously falling altitude. The landing section closes the mission loop.

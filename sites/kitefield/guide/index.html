Kitefield

A kite festival in verlet rope and canvas sky

The Brief

What Was Asked

kitefield — a kite festival. The one daylight-and-wind site. Signature: verlet rope-and-cloth kites in a procedural sky — fly one with your pointer, wind gusts ripple tails and grass; the crowd of kites is autonomous. Palette: sky #d8e9f2, ink #21374a, kite red #e34234, string #7a6f5d. Type: Bricolage Grotesque + Atkinson Hyperlegible. Prove: soft-body physics with joy; a bright palette that still has hierarchy. Upgrade ideas: wind sock and forecast; kite designer (pick sail pattern); tangled-lines easter egg.

The brief demanded physics that felt joyful — not technically correct for its own sake, but expressively correct. That meant solving why kites hang below their anchor (string gravity dominating kite lift) before worrying about anything visual.

Signature Technique

Near-weightless Verlet Strings

The core physics insight: a standard verlet string particle applies full gravity to every segment, which drags the kite body below its anchor point. The fix was making string particles nearly weightless while giving the kite end full upward lift:

// Physics tuning: string is nearly weightless (blown by wind),
// kite has strong lift
const STR_GRAVITY = GRAVITY * 0.03;  // 6 px/s² — near-weightless string
const KITE_LIFT   = -620;            // px/s² upward
// Net kite force: GRAVITY + KITE_LIFT = 200 - 620 = -420 upward
// String total sag: 16 segs × 6 px/s² = 96 px/s² — kite (-420) overcomes it

// Per-instance segs parameter gives natural altitude spread:
const crowd = [
  { rx: 0.03, segs:  9 },  // low-flying near-left
  { rx: 0.08, segs: 14 },  // high flier
  { rx: 0.22, segs: 17 },  // very high
  // shorter segs → lower kite; longer → higher
];

Bezier-Splined String Drawing (Pass 2 upgrade)

String segments are verlet particles connected by distance constraints — so the raw path is a polyline through discrete points. Replacing lineTo with midpoint-averaged quadratic bezier curves makes the string read as a single continuous silk drape:

// Midpoint-averaged quadratic bezier: smooth through all verlet particles
ctx.beginPath();
ctx.moveTo(this.sp[0].x, this.sp[0].y);
for (let i = 1; i < this.segs; i++) {
  const mx = (this.sp[i].x + this.sp[i + 1].x) * 0.5;
  const my = (this.sp[i].y + this.sp[i + 1].y) * 0.5;
  ctx.quadraticCurveTo(this.sp[i].x, this.sp[i].y, mx, my);
}
ctx.lineTo(this.sp[this.segs].x, this.sp[this.segs].y);

The same technique is applied to the kite tail ribbon for a fluid, fabric-like movement.

Swallow Silhouettes (Pass 3 upgrade)

Four stylised birds cross the sky at varied speeds and altitudes. Each is two quadratic bezier wing arcs with a gentle sinusoidal flap — the same bezier idiom as the strings, unified across the system:

function drawBird(bx, by, span, wingT) {
  const flap = Math.sin(wingT) * 0.28;
  const liftL = flap * span * 0.9;
  ctx.beginPath();
  // Left wing arc
  ctx.moveTo(bx, by);
  ctx.quadraticCurveTo(bx - span * 0.55, by - span * 0.3 - liftL, bx - span, by - liftL);
  // Right wing arc
  ctx.moveTo(bx, by);
  ctx.quadraticCurveTo(bx + span * 0.55, by - span * 0.3 - liftL, bx + span, by - liftL);
  ctx.stroke();
}
Iteration History

What Each Pass Actually Found

  • Pass 1
    Initial build and physics fix. The root problem: all kites descended below the grass line because GRAVITY (200 px/s²) dominated the string chain. Fixed by per-segment STR_GRAVITY = GRAVITY × 0.03 for string particles and KITE_LIFT = −620 px/s² for the kite end. Per-instance segs parameter (8–17) gives natural altitude spread. Complexity upgrade: directional sun glow (radial gradient from upper-right). Also added: wind sock with Beaufort label, animated grass (200 cubic-bezier blades), 5 cumulus cloud clusters drifting with wind, responsive layout (wind sock hidden at W < 520, title repositions on mobile).
  • Pass 2
    Bezier spline strings + wind calibration. Screenshot critique found: string polylines read as rigid ladder segments rather than continuous silk. Fix: midpoint-averaged quadratic bezier through all verlet particles for both string and tail. Wind base lowered from 72 → 50 px/s² so "Light Breeze" appears by default instead of nearly always "Moderate". Bow-tie tail ribbons widened at small kite scales (bowSize scales with bodyScale inverse). Zero console errors after changes.
  • Pass 3
    Swallow silhouettes + atmospheric life. Complexity upgrade: four small birds with sinusoidal wing-beat cross the sky at varied speeds and altitudes. Each is two quadratic bezier arcs — the same curve idiom as the strings, keeping the system visually unified. Birds reset off-screen left with new random altitude when they exit right. Applied to reduced-motion static frame via warmup loop. Zero console errors, zero page errors at all viewports.
Honest Inventory

What Remains Rough

The string still has no true catenary sag — physics gives a reasonable approximation but a real mathematical catenary would add more elegant droop in calm conditions. The kite body is a rigid diamond; under wind pressure it would deform. The wind sock occasionally shows "Moderate" even at the lower base wind because oscillation and gust layers can push total wind force above the 55 px/s² threshold. None of these are release blockers, and fixing them risks making the simulation feel engineered rather than joyful.

Brief upgrades not built: kite designer (pick sail pattern), tangled-lines easter egg. Both need non-trivial interaction state machines that were de-scoped for single-agent budget.
